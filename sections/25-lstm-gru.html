为解决 RNN 的梯度消失问题,1997 年 LSTM 横空出世,2014 年 GRU 做了精简版。它们的核心是"门控" —— 让网络学会主动决定"该记什么、该忘什么"。

LSTM 的核心思想

一句话 普通 RNN 只有一个状态 $h$。LSTM 额外引入一个"细胞状态" $C$,像高速公路一样贯穿整个序列,几乎不受非线性衰减。 然后用三个门精确控制信息如何进出这条高速公路。

三个门 + 细胞状态

$$f_t = \sigma(W_f[h_{t-1}, x_t] + b_f) \qquad \text{遗忘门}$$ $$i_t = \sigma(W_i[h_{t-1}, x_t] + b_i) \qquad \text{输入门}$$ $$o_t = \sigma(W_o[h_{t-1}, x_t] + b_o) \qquad \text{输出门}$$ $$\tilde{C}_t = \tanh(W_C[h_{t-1}, x_t] + b_C) \qquad \text{候选细胞}$$ $$C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t \qquad \text{细胞更新(加法!)}$$ $$h_t = o_t \odot \tanh(C_t)$$

每个门输出是 $(0,1)$ 之间的向量,像可学习的开关:"0 = 关闭、1 = 全通过"。

为什么"加法更新"让梯度畅通

关键公式 $C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t$ 是加法。算 $\partial C_t / \partial C_{t-1}$:

$$\frac{\partial C_t}{\partial C_{t-1}} = f_t$$

这是一个元素级的值,如果 $f_t$ 接近 1,梯度几乎 100% 通过。普通 RNN 的对应项是 $W_{hh}^T \cdot \text{diag}(\tanh'(\cdot))$,会指数级衰减。加法结构是 LSTM 对抗梯度消失的数学原因。

GRU(Gated Recurrent Unit)

Cho 等人 2014 年提出。LSTM 的精简版 —— 合并细胞状态和隐藏状态、把 3 个门减到 2 个(更新门 z + 重置门 r)。参数少 25% 左右,效果接近 LSTM。

$$z_t = \sigma(W_z[h_{t-1}, x_t]) \qquad \text{更新门}$$ $$r_t = \sigma(W_r[h_{t-1}, x_t]) \qquad \text{重置门}$$ $$\tilde{h}_t = \tanh(W[r_t \odot h_{t-1},\; x_t])$$ $$h_t = (1-z_t)\odot h_{t-1} + z_t\odot \tilde{h}_t$$

GRU 视图二:单个时间步内部

RNN · LSTM · GRU 对比

RNNLSTMGRU
状态数1 (h)2 (h, C)1 (h)
门数032
参数量1×4×3×
长距离依赖差(<10 步)极强(数百步)强
训练速度最快最慢中等
2026 年的现实 NLP 主流任务已被 Transformer 取代,但 LSTM/GRU 在以下场景仍有价值:
  • 小数据集(Transformer 需要大量数据)
  • 低延迟 / 流式 / 边缘设备部署
  • 某些时间序列预测(股票、传感器)
  • 强化学习的 recurrent policy

PyTorch 代码

import torch.nn as nn

# LSTM
lstm = nn.LSTM(input_size=64, hidden_size=128, batch_first=True, num_layers=2)
x = torch.randn(32, 10, 64)          # (batch, seq, feature)
output, (h_n, c_n) = lstm(x)          # output: (32, 10, 128)

# GRU —— 完全一样的接口,少个 C
gru = nn.GRU(input_size=64, hidden_size=128, batch_first=True)
output, h_n = gru(x)

# 双向 LSTM(BiLSTM)
bilstm = nn.LSTM(64, 128, bidirectional=True, batch_first=True)
# output 维度翻倍:(32, 10, 256)
