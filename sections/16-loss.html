损失函数回答一个问题:模型预测 $\hat{y}$ 和真实 $y$ 差多少? 选对损失函数,比选对架构还重要。

两大主流:MSE(回归) vs CE(分类)

均方误差(MSE)

$$L_{MSE} = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2$$

预测连续值(房价、温度、股价...)。平方惩罚让偏差越大惩罚越狠。

交叉熵(Cross Entropy)

$$L_{CE} = -\sum_i y_i \log \hat{y}_i$$

分类任务。若真实类是 one-hot $\mathbf{y}=[0,1,0]$,公式简化为 $L = -\log\hat{y}_{true}$ —— 预测真实类的概率越低,惩罚越大(因为 $-\log$ 在 0 附近爆炸)。

二元交叉熵(BCE) — 二分类专用

$$L_{BCE} = -y\log\hat{y} - (1-y)\log(1-\hat{y})$$

二分类时,Sigmoid 输出 $\hat{y} \in (0,1)$ 表示"属于正类的概率",配合 BCE 就是"黄金组合"。 PyTorch 里用 nn.BCELoss() 或更稳定的 nn.BCEWithLogitsLoss()(内置 Sigmoid)。

🌟 Softmax + 交叉熵 合并求导的小奇迹

多分类时网络最后一层输出 logits $\mathbf{z}$,经 Softmax 得到 $\hat{\mathbf{y}}$,再算 CE 损失。 单独求 CE 对 Softmax 的导数很复杂,但合起来导数非常简洁:

$$\boxed{\;\frac{\partial L}{\partial z_i} = \hat{y}_i - y_i\;}$$

仅仅是"预测概率 - 真实 one-hot"!这个美到令人发指的公式,是面试高频考点,也是 PyTorch nn.CrossEntropyLoss() 内部合并实现的原因(数值稳定 + 计算高效)。

注意 用 nn.CrossEntropyLoss() 时,网络最后一层不要再加 Softmax! 它内部已经把 Softmax 和 CE 合并计算了(log-sum-exp 技巧)。 输出 raw logits 就行。

其他损失

名称公式场景
MAE(L1 损失)$|y-\hat{y}|$对异常值鲁棒(比 MSE 更"稳")
Huber小误差 MSE,大误差 MAE两者折中
Focal Loss$-(1-\hat{y})^\gamma \log\hat{y}$类别严重不平衡的检测
对比损失相似对拉近,不同对推远CLIP / 自监督学习
KL 散度$\sum y_i \log\frac{y_i}{\hat{y}_i}$分布对齐(VAE / 知识蒸馏)

选择指南

  • 连续回归 → MSE(有异常值) → MAE 或 Huber
  • 二分类 → Sigmoid + BCE(或 logits + BCEWithLogitsLoss)
  • 多分类 → 不加激活 + CrossEntropyLoss
  • 极端类别不平衡(检测任务) → Focal Loss
  • 分布拟合 / 蒸馏 → KL 散度
