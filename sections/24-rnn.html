图像是固定大小,但句子、音频、股价是序列,长度可变。RNN 为此而生。 核心想法:有一个隐藏状态 $h$,像"大脑的记忆",每读一个新输入就更新一次,并参与下一步计算。

核心公式

$$h_t = \tanh(W_{xh}\,x_t + W_{hh}\,h_{t-1} + b_h)$$ $$y_t = W_{hy}\,h_t + b_y$$

注意:三个权重 $W_{xh}, W_{hh}, W_{hy}$ 在每个时间步都是同一个(参数共享),这是 RNN 能处理可变长度序列的关键。

三种使用配置

类型输入→输出典型例子
一对多一个 → 序列看图写文(image captioning)
多对一序列 → 一个情感分析、文本分类
多对多(同步)序列 → 等长序列词性标注
多对多(不同步)序列 → 不等长机器翻译(Seq2Seq,Encoder-Decoder)

双向 RNN(BiRNN)

普通 RNN 只看过去。对某些任务(如命名实体识别),未来上下文同样重要。双向 RNN 同时跑一个正向 RNN 和一个反向 RNN,把两个方向的隐藏状态拼接:

$$h_t = [\overrightarrow{h}_t;\; \overleftarrow{h}_t]$$

Teacher Forcing — 训练 Seq2Seq 的技巧

训练翻译模型时,解码器应该把自己的上一步输出喂给下一步。但初期模型预测很差,错误会级联放大。 解决:训练时直接把真实的 $y_{t-1}$ 喂进去(不管当前预测是啥),叫 Teacher Forcing。推理时才真正用自己的预测。

⚠️ 梯度消失 / 爆炸

RNN 展开后相当于很深的网络。反向传播时,梯度要连乘 $W_{hh}$ 很多次:

$$\frac{\partial L}{\partial h_1} \propto \prod_{t=1}^{T} W_{hh}^T \cdot \text{diag}(\tanh'(\cdot))$$
  • 若 $\|W_{hh}\| < 1$ → 梯度指数级衰减(消失) → 早期输入无法影响后期
  • 若 $\|W_{hh}\| > 1$ → 梯度指数级增长(爆炸) → 训练发散

朴素 RNN 一般只能记住 10-20 步 的信息。要记更久?LSTM / GRU 登场。

PyTorch 代码

import torch.nn as nn

class SimpleRNN(nn.Module):
    def __init__(self, vocab_size, hidden_size):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, 64)
        self.rnn   = nn.RNN(input_size=64, hidden_size=hidden_size, batch_first=True)
        self.fc    = nn.Linear(hidden_size, vocab_size)

    def forward(self, x):
        emb = self.embed(x)              # (B, T, 64)
        out, h = self.rnn(emb)           # out: (B, T, H)
        return self.fc(out)              # (B, T, vocab)
