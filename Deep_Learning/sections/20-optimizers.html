普通 SGD 会震荡、陷入鞍点。现代优化器加了"物理技巧"来加速和稳定。 Adam、AdamW 是当今 99% 场景的默认选择。

四个主流优化器

名称核心思想公式
SGD直接按梯度走$\mathbf{w} \mathrel{-}= \eta\nabla L$
Momentum加"惯性"平滑方向$\mathbf{v} = \beta\mathbf{v} + \nabla L, \mathbf{w} \mathrel{-}= \eta\mathbf{v}$
Nesterov先预走一步再算梯度"先看一眼下一步位置的梯度"
Adam动量 + 每参数自适应学习率维护一阶 $m$ 和二阶 $v$ 动量

Adam 公式(现代 DL 的默认)

$$m_t = \beta_1 m_{t-1} + (1-\beta_1)\,\nabla L \qquad \text{(一阶动量 ≈ 梯度平均)}$$ $$v_t = \beta_2 v_{t-1} + (1-\beta_2)\,(\nabla L)^2 \qquad \text{(二阶动量 ≈ 梯度平方平均)}$$ $$\hat{m}_t = \frac{m_t}{1-\beta_1^t}, \quad \hat{v}_t = \frac{v_t}{1-\beta_2^t} \qquad \text{(偏差修正)}$$ $$\mathbf{w} \mathrel{-}= \frac{\eta}{\sqrt{\hat{v}_t}+\epsilon}\,\hat{m}_t$$

默认超参 $\beta_1=0.9, \beta_2=0.999, \epsilon=10^{-8}$,调一下 $\eta$ 就能跑大部分任务。

Adam 为什么最常用

  1. 动量让梯度方向"平滑",减少震荡
  2. 自适应学习率:梯度小的维度加速,梯度大的维度减速,不同维度获得相似相对更新幅度
  3. 超参稳定,几乎开箱即用

AdamW — Transformer 时代的标配

Adam 直接做 L2 正则时有个问题:权重衰减被自适应学习率扭曲。AdamW(Loshchilov 2017)把权重衰减和梯度更新解耦:

$$\mathbf{w} \mathrel{-}= \eta\!\left(\frac{\hat{m}_t}{\sqrt{\hat{v}_t}+\epsilon} + \lambda\,\mathbf{w}\right)$$

其中 $\lambda$ 是独立的 weight decay。现代 Transformer(BERT, GPT, LLaMA)训练几乎清一色用 AdamW。

Nesterov 动量的直觉

普通动量:先按动量"冲",再算当前位置梯度。Nesterov:先按动量冲到"未来位置",然后算那里的梯度,更有前瞻性 —— 像球滚下山时"提前看见"要撞到的墙。

选择指南

实用决策树
  • 新手 / 默认 → Adam(lr=3e-4)(Karpathy 的"3e-4 常数")
  • 大模型 / Transformer → AdamW(lr=1e-4, wd=0.01)
  • CNN 追求极致 → SGD(momentum=0.9) + cosine scheduler 常能超过 Adam
  • 内存紧张 → SGD(不用存一阶二阶动量,省一半显存)

PyTorch 代码

import torch.optim as optim

# SGD
opt = optim.SGD(model.parameters(), lr=0.01)

# SGD + Momentum
opt = optim.SGD(model.parameters(), lr=0.01, momentum=0.9, nesterov=True)

# Adam(最常用)
opt = optim.Adam(model.parameters(), lr=3e-4)

# AdamW(Transformer 标配)
opt = optim.AdamW(model.parameters(), lr=1e-4, weight_decay=0.01)
