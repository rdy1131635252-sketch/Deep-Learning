过拟合 = 模型把训练数据背下来、但泛化差。L1/L2 正则 · BN · LayerNorm · Dropout · Early Stopping 是抗过拟合工具箱。

L1 vs L2 正则化

$$L_{total} = L_{data} + \lambda \underbrace{\sum_i w_i^2}_{\text{L2}}\quad \text{或} \quad L_{total} = L_{data} + \lambda \underbrace{\sum_i |w_i|}_{\text{L1}}$$

几何直觉:菱形 vs 圆形约束

图 3.9.1 损失的椭圆等高线在约束域内尽量靠近中心。L1 菱形有尖角 → 稀疏(某些权重被精确置零);L2 圆形光滑 → 权重小但通常非零。

公式$\sum|w_i|$$\sum w_i^2$
对大权重惩罚线性增长平方增长 → 更狠

Batch Normalization(BN)

$$\hat{x} = \frac{x - \mu_{batch}}{\sqrt{\sigma^2_{batch} + \epsilon}}, \qquad y = \gamma\hat{x} + \beta$$

对一个 batch 的同一维度做归一化 → 变成 0 均值 1 方差 → 再用 $\gamma, \beta$ 学回来(保留表达能力)。

BN 的好处
  • 稳定训练(每层分布不再漂移)
  • 允许更大学习率
  • 一定的正则化效果(batch 统计带来随机性)
  • 对初始化不敏感
BN 的坑 训练和推理行为不同:训练用 batch 统计,推理用训练时的滑动平均。 PyTorch 里必须在推理前调用 model.eval() 切换,否则模式错乱。

LayerNorm — Transformer 的标配

Transformer 为什么不用 BatchNorm,改用 LayerNorm?两点原因:

  1. 序列长度不定:BN 需要对每个时间步独立统计,LN 只对特征维度做,长度无所谓
  2. Batch 依赖 → 不稳:BN 在小 batch / 推理时效果飘,LN 不依赖 batch 大小,推理和训练完全一致
$$\text{LayerNorm}(\mathbf{x}) = \gamma\,\frac{\mathbf{x}-\mu}{\sqrt{\sigma^2+\epsilon}} + \beta \quad \text{其中 $\mu, \sigma$ 对 $\mathbf{x}$ 的所有维度计算}$$

Dropout

训练时随机把一部分神经元"熄灭"(设为 0),强迫网络不依赖某几个神经元,学到更鲁棒的特征。 推理时不熄灭(但要缩放,或训练时用 Inverted Dropout 先缩放)。

每次"重采样",不同的神经元被熄灭(红叉)。训练时网络每一步看见的都是"子网络"。

Early Stopping — 最简单的正则

监控验证集损失,一旦连续若干轮不再下降,就停止训练。几行代码,却效果立竿见影。

正则化家族的本质 所有这些方法,本质都在"限制模型的自由度" —— L1/L2 限制权重范数,Dropout 随机屏蔽参数,BN/LN 限制激活分布,Early Stopping 限制训练时长。限制得当 → 学到泛化能力,过度限制 → 欠拟合。

PyTorch 代码

import torch.nn as nn

net = nn.Sequential(
    nn.Linear(784, 256),
    nn.BatchNorm1d(256),        # BN
    nn.ReLU(),
    nn.Dropout(0.3),             # 30% 的 Dropout
    nn.Linear(256, 10)
)

# L2 正则(weight decay)在优化器里
opt = torch.optim.AdamW(net.parameters(), lr=1e-3, weight_decay=1e-4)

# 训练时
net.train()
# 推理时(关闭 Dropout,BN 用滑动平均)
net.eval()
