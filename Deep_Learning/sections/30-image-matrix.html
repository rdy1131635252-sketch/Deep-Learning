计算机眼里的"图像"就是一组数字 —— 一张矩阵。理解这一点,你就理解了整个 CV 的输入。

灰度图 = 二维矩阵

每个像素一个 0-255 的数(0 = 黑,255 = 白)。一张 28×28 的 MNIST 数字图就是一个 $28\times 28$ 的矩阵:

在左侧画一个数字 → 右侧实时显示像素值矩阵

像素矩阵 (28×28)
互动 8.1 在左边写一个数字 → 点像素化 → 看到的就是网络真正"看到"的内容。

彩色图 = 三维张量

RGB 图像每像素 3 个值(红绿蓝),整体形状 $(H, W, 3)$ 或 PyTorch 习惯的 $(3, H, W)$。一张 224×224 彩色图 = $3 \times 224 \times 224 = 150{,}528$ 个数字。

$$\text{Image}_{\text{color}} \in \mathbb{R}^{C \times H \times W}, \quad C \in \{1, 3, 4\}$$

常见预处理

  • 归一化到 [0, 1]:除以 255
  • 标准化:减均值、除标准差(每通道独立)。ImageNet 上的常用值:
    mean = [0.485, 0.456, 0.406], std = [0.229, 0.224, 0.225]
  • resize:把图改到固定大小(如 224×224 喂 ResNet)
  • 转 tensor:HWC → CHW,uint8 → float32

批量形状

训练时一批 $B$ 张图 → 4D 张量 $(B, C, H, W)$。一次前向把一整批图喂给网络。

$$\text{Batch} \in \mathbb{R}^{B \times C \times H \times W}, \quad \text{典型 32 \times 3 \times 224 \times 224}$$

PyTorch 代码

from PIL import Image
import torch
from torchvision import transforms

img = Image.open('cat.jpg').convert('RGB')

transform = transforms.Compose([
    transforms.Resize((224, 224)),
    transforms.ToTensor(),                       # → (3, 224, 224), [0, 1]
    transforms.Normalize(
        mean=[0.485, 0.456, 0.406],
        std=[0.229, 0.224, 0.225]
    ),
])

tensor = transform(img)                         # (3, 224, 224)
batch  = tensor.unsqueeze(0)                    # (1, 3, 224, 224)
print(batch.shape, batch.min(), batch.max())
