模型架构定下来后,真正决定能不能训出好结果的是三件事:权重初始化、学习率调度、训练稳定性技巧。

权重初始化:为什么不能全设 0

如果所有权重都是 0,所有神经元的输出和梯度都相同 → 对称性破不开,网络等同于只有一个神经元。 设成全相同的常数也一样。所以必须随机初始化,但范围有讲究。

Xavier / Glorot 初始化(2010)

$$W \sim \mathcal{N}\!\left(0,\ \frac{2}{n_{in}+n_{out}}\right)$$

推导逻辑:让每一层激活值的方差保持稳定,既不爆炸也不消失。适合 sigmoid/tanh。

He / Kaiming 初始化(2015)

$$W \sim \mathcal{N}\!\left(0,\ \frac{2}{n_{in}}\right)$$

针对 ReLU 推导(ReLU 把一半激活变 0,所以方差比 Xavier 大一倍)。 用 ReLU 默认就用 He 初始化。PyTorch 的 nn.Linear 默认就是 Kaiming 风格。

学习率调度:开局慢、中段快、收官细

固定学习率往往不是最优。常用调度:

策略形状适合
Step decay每 N 轮乘 0.1简单,CV 经典
Cosine annealing余弦曲线下降SOTA CV 训练
Warmup + Cosine前几千步线性升,然后余弦降Transformer 标配
OneCycle升 → 降 → 谷底fast.ai 推荐
ReduceLROnPlateau验证集不降时手动减保险

为什么大模型一定要 warmup?

训练初期梯度方向不可靠。如果一开始就用大学习率,可能把好的初始化破坏掉。 warmup 让前几千步"试探着走",到稳定后再上大学习率。 对 LLM/Transformer 几乎是必须的。

梯度裁剪 — 防止训练崩溃

遇到梯度爆炸,简单粗暴的解决:限制梯度的范数,如果超了就缩放回去:

$$\mathbf{g} \leftarrow \mathbf{g} \cdot \min\!\left(1, \frac{\text{max\_norm}}{\|\mathbf{g}\|}\right)$$

典型阈值 1.0。RNN 训练几乎必加。Transformer 也常加。

标签平滑(Label Smoothing)

分类任务中,one-hot 标签 $[0,0,1,0,0]$ 会迫使模型输出极端概率(99.9%),容易过拟合, 也让模型对错误标签过于自信。把标签变软:

$$y' = (1-\epsilon) \cdot y + \frac{\epsilon}{K}, \quad \epsilon = 0.1$$

典型 $\epsilon=0.1$,真实类的概率从 1.0 → 0.92,其他类从 0 → 0.02。模型预测会更校准。

Mixed Precision Training(FP16/BF16)

默认 32 位浮点很慢且费显存。混合精度:权重和算梯度用 FP16/BF16,关键累加用 FP32 —— 内存减半、吞吐 1.5-2× 提升、几乎无精度损失。现代框架(PyTorch torch.cuda.amp、 DeepSpeed、bitsandbytes)开箱可用。

批大小的影响

  • 小 batch (32-64):梯度噪声大,有正则化效果,泛化常更好
  • 大 batch (1024+):吞吐高,但需要按比例放大学习率(linear scaling rule)
  • 梯度累积:显存不够想要大 batch?积累 N 步小 batch 的梯度再更新一次

PyTorch 代码

import torch.nn as nn
from torch.optim.lr_scheduler import CosineAnnealingLR, LambdaLR

# He 初始化(默认就是)
nn.init.kaiming_normal_(layer.weight, nonlinearity='relu')

# Warmup + Cosine 调度
def lr_lambda(step):
    if step < warmup_steps:
        return step / warmup_steps
    progress = (step - warmup_steps) / (total_steps - warmup_steps)
    return 0.5 * (1 + math.cos(math.pi * progress))

scheduler = LambdaLR(optimizer, lr_lambda)

# 梯度裁剪
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

# 标签平滑(PyTorch 1.10+)
loss = nn.CrossEntropyLoss(label_smoothing=0.1)

# 混合精度
from torch.cuda.amp import autocast, GradScaler
scaler = GradScaler()
with autocast():
    out = model(x)
    loss = criterion(out, y)
scaler.scale(loss).backward()
scaler.step(optimizer)
scaler.update()
